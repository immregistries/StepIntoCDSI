Step Into CDSi

Step Into CDSi is an open-source reference and demonstration tool for understanding how immunization clinical decision support (CDS) logic is executed in practice. It enables developers, analysts, and public-health stakeholders to explore vaccine evaluation and forecasting behavior in a transparent, step-by-step manner rather than relying on opaque “black-box” forecasting engines.

This project is independent of the CDC CDSi initiative, but it is designed to work with and illuminate the publicly available CDSi Logic Specification, Supporting Data, and test materials published by CDC. For official CDSi background and resources, see: https://www.cdc.gov/iis/cdsi/index.html

Application Areas

Step

Walk through CDS logic one step at a time for a specific patient scenario. Each step corresponds to a chapter of the CDSi Logic Specification, allowing users to see exactly how intermediate decisions and calculations are produced.

Supporting Data

Browse and inspect the CDSi supporting data used by the engine. This enables direct comparison between specification inputs and step-level processing behavior.

Forecast

Generate a full text forecast similar to outputs produced by production CDS engines. The system evaluates immunization history, applies CDSi logic end-to-end, and returns final evaluation and recommendation results.

FHIR Server

Expose forecasting functionality through a FHIR endpoint aligned with the Immunization Decision Support (ImmDS) FHIR implementation guide: https://hl7.org/fhir/us/immds/index.html

FHIR Sandbox

Provide a simple FHIR client for interacting with the FHIR server, supporting testing, validation, and demonstration of CDS interoperability workflows.

Transparent Immunization Decision Support

Immunization clinical decision support converts complex vaccination guidance into computable logic that determines which vaccines a patient needs and when they should be given. Because national recommendations are published in clinical language rather than executable rules, implementations across health information systems often vary. Step Into CDSi makes this translation visible by exposing each stage of evaluation and forecasting, allowing users to trace how patient history, timing rules, and schedule logic combine to produce final recommendations.

Designed for Learning, Testing, and Demonstration

Step Into CDSi is intentionally built as a reference and demonstration system, not a production clinical service. The application enables step-through inspection of logic, review of supporting data, and generation of realistic forecast output comparable to operational CDS engines. As of February 2025, approximately 90% of CDSi-defined test cases pass, though patient condition logic and certain edge scenarios remain incomplete. Results are broadly representative of real-world system behavior and useful for education, prototyping, and interoperability demonstrations, but the system must not be used for clinical or public-health decision making.

Integration with the NIST FITS Testing Platform

Step Into CDSi works directly with the Forecasting for Immunization Test Suite (FITS) maintained by NIST: https://fits.nist.gov/fits/. CDC publishes official CDSi test cases through FITS, and the platform supports bidirectional testing workflows. Step Into CDSi can retrieve and execute FITS test cases locally for validation and debugging, while FITS can also invoke Step Into CDSi as an external forecasting engine for comparative testing. These complementary patterns support specification verification, regression testing, and interoperability experimentation across CDS implementations.

Open Source Java Implementation

The full source code is publicly available on GitHub: https://github.com/immregistries/StepIntoCDSI. Step Into CDSi is implemented as a Java web application designed to run in a standard servlet container such as Apache Tomcat and is distributed under the Apache 2.0 open-source license. This allows implementers, researchers, and standards developers to review the logic, adapt the code, and use the system as a foundation for experimentation or education without licensing barriers.

Access and Security Context

The demonstration application itself does not require login or authentication for general use. However, integration with the FITS platform for automated testing requires valid FITS credentials issued through that service. This separation preserves ease of demonstration while respecting the access controls associated with official test infrastructure.
